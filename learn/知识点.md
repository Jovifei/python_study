在Python中，字符串可以用单引号(')、双引号(")或三引号("""或''')来表示。
问题：
为什么以下代码表现不同？
python

编辑

print("""hello world""")
python

编辑

print("hello world")
第一个可以使用三个"""，第二个使用一个"可以打印，但是
python

编辑

print(""hello world"")
编译失败？
答：
print("""hello world""") - 这里使用了三重双引号，可以包含多行字符串，并且不需要转义内部的引号。
print("hello world") - 这里使用了单重双引号，可以包含普通的单行字符串。
print(""hello world"") - 这种写法会失败，因为Python解析器会这样理解：
第一个双引号 "" 开始一个字符串
然后遇到 hello world
最后又遇到一个双引号 ""，这会让Python认为字符串已经结束，但后面还有一个额外的引号，导致语法错误



## 2除法运算
在Python 3中，/ 运算符执行的是浮点除法（true division），即使两个整数相除，结果也会是浮点数类型。
print(4/2) 输出 2.0 而不是 2，这是因为Python 3中除法运算的结果默认是浮点数。
同样地，print(10/3) 会输出 3.3333333333333335，这是因为：
10除以3在数学上是3.333...（无限循环小数）
计算机使用浮点数表示这个值，受到二进制浮点数精度限制
所以显示为3.3333333333333335（64位浮点数精度下最接近的表示）
如果你想得到整数除法结果，可以使用 // 运算符：
print(10//3) 输出 3，因为 // 运算符执行的是整数除法（floor division），结果会向下取整到最接近的整数。

# 身份比较运算符
在Python中，身份比较运算符 is 和 is not 用于比较两个对象的内存地址是否相同。如果两个对象的内存地址相同，那么它们就是同一个对象，否则它们是不同的对象。
a = [1, 2, 3]
b = a           # b 指向和 a 同一个列表对象
c = [1, 2, 3]   # c 是一个新创建的、内容相同的列表

print(a is b)      # True —— a 和 b 是同一个对象
print(a is c)      # False —— a 和 c 内容相同但不是同一个对象
print(a == c)      # True —— 值相等（但不是同一个对象）
